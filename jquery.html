Encadenamiento de métodos


La función jQuery wrapper siempre devuelve el conjunto de elementos con los que coincide, por lo que puedes volver a operar sobre ella, si quieres. Saber que el wrapper de jQuery siempre devuelve algo es muy útil para encadenar métodos de jQuery. Esto se hace para evitar almacenar los resultados en variables temporales, o repetir la búsqueda de un conjunto de coincidencias. Una trampa


La práctica común durante muchos años ha sido poner el código JS en el HEAD de una página web, pero eso nos tiende una trampa. Si escribimos algún código JQ para encontrar un elemento hacerle algo, como hemos hecho muchas veces en esta lectura, pero el HTML para ese elemento está más abajo en la página, el elemento no existirá cuando el código JQ se ejecute. Aquí hay un ejemplo:


News for Today

Nuestra intención es bastante clara: encontrar el H1 e insertar la fecha de hoy en él. Sin embargo, cuando el código JavaScript/jQuery se ejecuta, el H1 todavía no existe, por lo que no se rellenará con la fecha de hoy.
Y lo que es peor, ¡no recibiremos un mensaje de error! jQuery ha encontrado con éxito un conjunto de elementos; simplemente ha sido un conjunto vacío, lo que está bien para jQuery, pero no para nosotros.

¿Está listo el DOM?


la trampa que acabamos de describir nos lleva a la importante cuestión de cuándo debe ejecutarse nuestro código jQuery. Poner el código en diferentes partes del archivo HTML puede tener diferentes efectos en la página, según el momento en que el navegador lea e interprete el código. Por eso, normalmente, el código jQuery debe ser ejecutado sólo después de que el DOM haya sido creado y esté listo para la manipulación dinámica.
La mayoría de los ejemplos que verás en la lectura proporcionada, mostrarán código normalmente envuelto en un gran manejador de eventos para el objeto documento, como se muestra a continuación:



News for Today


Esto funciona correctamente, pero es complejo de entender. Utiliza la función envolvente de jQuery con el objeto documento especial y el resultado obtiene el método ready() se dispara cuando el DOM está listo), y a este método le pasa como argumento una función anónima. También es una pesadilla sintáctica: fíjate que la última línea es });
Otra forma de asegurarse de que el DOM está listo es poner el código JavaScript al final de la página (justo antes de (body) en lugar de en el head. De esta manera, sabemos que las líneas HTML anteriores ya han sido procesadas por el navegador y el DOM está listo.
